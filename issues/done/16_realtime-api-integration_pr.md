# 実際の水位・気象データ取得への切り替え（モックの廃止）

## 対象 Issue
Issue id: 16

## 変更内容
- 国交省（または「川の防災情報」などから取得可能な公開情報）のデータソースから最新の水位情報を取得する `RiverApiService` を実装しました。
- 気象庁（アメダスJSONデータなど）のデータソースから最新の気象情報を取得する `JmaApiService` を実装しました。
- `app:poll-water-level` コマンド内のモックデータ生成ロジックを廃止し、`RiverApiService` を利用して実データを取得するように改修しました。APIからデータを取得できない場合はSqsに送信しないようにしました。
- `app:poll-weather` コマンド内のモックデータ生成ロジックを廃止し、`JmaApiService` を利用して実データを取得するように改修しました。APIからデータを取得できない場合はSqsに送信しないようにしました。
- 各種テストを修正し、全てのテストが通ることを確認しました。

## 確認事項
- `RiverApiService`, `JmaApiService` 内のAPIリクエストエラー時は null を返し、Poller 側で適切にスキップされることを確認しました。
- 構文エラーがなく、ローカルテスト（Pest）が正常にパスすることを確認しました。
- Laravel Pint によるコード整形を適用しました。
